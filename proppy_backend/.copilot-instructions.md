# 🧠 Copilot Instructions — Proppy Backend (Django + DRF)

## 🎯 Goal

Build a clean, modular, and KISS backend for a property management app.

---

## ⚙️ Stack

- Django
- Django REST Framework (DRF)
- Custom User model (email as username)
- JWT authentication (simplejwt)

---

## 🧱 Architecture Principles

- Keep everything **KISS (Keep It Simple and Stupid)**
- Prefer **APIView** over generic views unless explicitly needed
- Avoid unnecessary abstractions
- Do NOT introduce complexity (services, managers, etc.) unless requested

---

## 📦 File Structure Rules

Each app must contain:

- `models.py`
- `views.py`
- `serializers.py`
- `urls.py`

Optional:
- `permissions.py` (only if needed)

---

## 🔌 API Design Rules

- Use `APIView` for custom logic (default choice)
- Use `ListAPIView` / `RetrieveAPIView` only for simple read endpoints
- Do NOT use `ModelSerializer` for complex flows (e.g. register, password reset)
- Always validate request data manually for MVP

---

## 🔐 Auth & Security

- Always use `get_user_model()` (never import User directly)
- Never expose sensitive information (e.g. email existence)
- Always validate tokens and permissions explicitly
- Use `set_password()` for password updates

---

## 🧠 Code Style

- Write clear and short docstrings:
  - WHAT the endpoint does
  - WHY it exists
- Add comments only where logic is not obvious
- Keep functions small and readable

---

## 🚫 Avoid

- Overengineering
- Premature optimization
- Complex abstractions
- Nested serializers unless explicitly needed

---

## 🧪 Testing

- Use pytest
- Write basic tests for:
  - happy path
  - invalid input
  - permission checks

---

## 🎯 MVP Mindset

- First make it work
- Then make it clean
- Then improve if needed

---

## 🧭 Naming

- Views: `SomethingView`
- Serializers: `SomethingSerializer`
- URLs: kebab-case (`password-reset-confirm/`)

---

## 🔁 Workflow

For each feature:

1. Create APIView
2. Add URL
3. Test manually (Postman / Insomnia)
4. Add pytest
5. Refactor if needed

---

## 💡 Important

If unsure:
- choose simpler solution
- ask for clarification instead of guessing
