# 🧠 Copilot Instructions — Proppy Frontend (Next.js + Tailwind)

## 🎯 Goal

Build a clean, minimal, and mobile-friendly UI for a property management app.

---

## ⚙️ Stack

- Next.js (App Router)
- React
- Tailwind CSS
- JWT auth (localStorage)

---

## 🧱 Architecture Principles

- Keep everything **KISS**
- Avoid overengineering
- Prefer simple components over abstractions
- Do NOT introduce unnecessary state management (Redux, Zustand, etc.)

---

## 📁 Folder Structure (IMPORTANT)

We use App Router with grouped routes:

- `(public)` → landing page
- `(auth)` → login, register
- `/dashboard` → protected area

Follow existing structure. Do NOT create new patterns.

---

## 🔌 Component Rules

- One component = one responsibility
- Keep components small and readable
- Avoid deeply nested components
- Avoid prop drilling → but also avoid over-abstracting

---

## 🎨 Styling (CRITICAL)

- Use Tailwind ONLY
- Mobile-first approach
- Use `max-w-*` for layout consistency
- Reuse global styles from `globals.css`
- Do NOT hardcode random colors — use existing tokens

---

## 🔐 Auth Handling

- Token stored in `localStorage`
- Use existing `apiClient`
- Do NOT reinvent auth logic
- Redirect after login/logout handled simply

---

## 📡 API Calls

- Use existing `apiClient` or `apiPublic`
- Do NOT call fetch directly unless explicitly needed
- Keep API calls simple (no abstraction layers)

---

## 🧠 Code Style

- Use functional components only
- Keep logic inline unless it grows
- Add short comments:
  - WHAT component does
  - WHY it exists

---

## 🚫 Avoid

- Overengineering
- Custom hooks unless necessary
- Global state managers
- Complex folder structures
- Premature optimization

---

## 🧪 UX Rules

- Always show loading state
- Always handle error state
- Keep forms simple
- Mobile-first layout

---

## 🎯 MVP Mindset

- First make it work
- Then make it clean
- Then improve UX

---

## 🧭 Naming

- Pages: `page.tsx`
- Layouts: `layout.tsx`
- Components: `PascalCase`
- Functions: `camelCase`

---

## 🔁 Workflow

For each feature:

1. Create page or component
2. Add UI (Tailwind)
3. Connect API
4. Test manually
5. Refactor if needed

---

## 💡 Important

If unsure:
- choose simpler solution
- do NOT invent new architecture
